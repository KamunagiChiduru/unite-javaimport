INSTALL                                             *unite-javaimport-install*

first you have to install

    1. Java Runtime Environment 1.6 or above
    1. Exuberant Ctags 5.8 or above

second you install this plugin. if you use |neobundle|, you can install this
plugin easily.
>
    " simple instalation
    NeoBundle 'KamunagiChiduru/unite-javaimport', {
    \   'depends': ['Shougo/unite.vim'],
    \}

    " fully instalation
    NeoBundle 'KamunagiChiduru/unite-javaimport', {

    \       'Shougo/unite.vim',
    \       'yuratomo/w3m.vim',
    \   ],
    \}

INTERFACE                                         *unite-javaimport-interface*

UNITE ~

    to show all importable classes by your classpath (by your .javaimport),
    execute |:Unite| command with "javaimport" as a source parameter.
>
    :Unite javaimport
<
    |unite-javaimport| can accept some arguments.

        ! or show_class
            show all classes.

        package
            a package name for filtering to gather classes.

        only
            a class name for filtering to gather classes.
            will gather only passed class name with different packages only.

COMMAND ~

    :JavaImportClearCache                              *:JavaImportClearCache*
        clear all cache.

    :JavaImportSortStatements                      *:JavaImportSortStatements*
        sort import statements on current buffer.

    :JavaImportRemoveUnnecessaries            *:JavaImportRemoveUnnecessaries*
        remove unnecessary import statements from current buffer.

MAPPING~

    <Plug>(javaimport-quickimport)            *<Plug>(javaimport-quickimport)*
        invoke |unite-javaimport| with "only" argument.

VARIABLE~

    g:javaimport_use_default_mapping        *g:javaimport_use_default_mapping*
        this variable type is a |Number|. if 1 has set |unite-javaimport| will
        set some mappings.

    g:javaimport_config                                  *g:javaimport_config*
        this variable type is a |Dictionary|. valid keys and effects are
        written below.

        cache_dir
            associated value type is a |String| and describes a directory will
            be cached some datas.

        preview_using
            associated value type is a |String| and must be a one of {w3m,
            lynx}. must be installed |w3m.vim| only if "w3m" was set, or
            "lynx" was set you got an error (since this feature is
            unimplemented yet).

HOW TO USE                                         *unite-javaimport-howtouse*

you have to write .javaimport to your current directory (project root
dictionary).

an example for .javaimport below.
>
    # line comment
    {
        'path':    'path to classpath',
    },
    {
        'path':    'path to classpath',
        'javadoc': 'javadoc uri for classpath',
    },
<
this is like a JSON. for each dictionary must have "path" and may have
"javadoc". details below.

    "path" (required)
        possible values are
            - src directory path
            - jar file path
            - zip file path
            - javadoc url
        a value accepts absolute and relative path.

    "javadoc" (optional)
        javadoc url
