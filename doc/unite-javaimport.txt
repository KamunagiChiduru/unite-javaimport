INSTALL                                             *unite-javaimport-install*

first you have to install

    1. Java Runtime Environment 1.7 or above
    1. Exuberant Ctags 5.8 or above

second you install this plugin. if you use |neobundle|, you can install this
plugin easily.
>
    " simple instalation
    NeoBundle 'KamunagiChiduru/unite-javaimport', {
    \   'depends': [
    \       'Shougo/unite.vim',
    \       'Shougo/vimproc.vim',
    \       'KamunagiChiduru/vim-javaclasspath',
    \       'KamunagiChiduru/vim-javalang',
    \   ],
    \}

    " fully instalation
    NeoBundle 'KamunagiChiduru/unite-javaimport', {
    \   'depends': [
    \       'Shougo/unite.vim',

INTERFACE                                         *unite-javaimport-interface*

Unite ~

    NOTE: A javaimport source is deprecated. Use javaimport/class source or
    other new sources instead of, please. A javaimport source will be removed
    in the future release.

    to show all importable classes by your classpath (by your .javaimport),
    execute |:Unite| command with "javaimport" as a source parameter.
>
    :Unite javaimport
<
    |unite-javaimport| can accept some arguments.

        ! or show_class
            show all classes.

        package
            a package name for filtering to gather classes.

        only
            a class name for filtering to gather classes.
            will gather only passed class name with different packages only.

                                                    *unite-javaimport-sources*
Unite Sources ~
                                               *unite-javaimport-source-class*
javaimport/class
    list all importable classes. it will uses |vim-javaclasspath| to detect
    your classpath.
>
    :Unite javaimport/class
<
Command ~

    :JavaImportClearCache                              *:JavaImportClearCache*
        clear all cache.

    :JavaImportSortStatements                      *:JavaImportSortStatements*
        sort import statements on current buffer.

    :JavaImportRemoveUnnecessaries            *:JavaImportRemoveUnnecessaries*
        remove unnecessary import statements from current buffer.

Mapping~

    <Plug>(javaimport-quickimport)            *<Plug>(javaimport-quickimport)*
        invoke |unite-javaimport| with "only" argument.
        (default key map: <Leader>I)

Variable~

    g:javaimport_use_default_mapping        *g:javaimport_use_default_mapping*
        this variable type is a |Number|. if 1 has set |unite-javaimport| will
        set some mappings.

    g:javaimport_config                                  *g:javaimport_config*
        this variable type is a |Dictionary|. valid keys and effects are
        written below.

        cache_dir
            associated value type is a |String| and describes a directory will
            be cached some datas.

        preview_using
            associated value type is a |String| and must be a one of {w3m,
            lynx}. must be installed |w3m.vim| only if "w3m" was set, or
            "lynx" was set you got an error (since this feature is
            unimplemented yet).

        server
            it's |Dictionary| value. keys are:

            vm
                it's |String| value. this is used to launch the
                javaimport-server application.
                (default on *nix:    $JAVA_HOME/bin/java)
                (default on windows: $JAVA_HOME/bin/javaw)
            host
                it's |String| value. it's a host name which javaimport-server
                is running or will be run.
                (default: localhost)
            port
                it's |Number| value. it's a port number which
                javaimport-server listening to or will be listen.
                (default: 51234)
            lockfile
                it's |String| value. it's a lockfile for javaimport-server.
                (default: g:javaimport_config.cache_dir . '/server.lock')

HOW TO USE                                         *unite-javaimport-howtouse*

configure classpath using |vim-javaclasspath|.
or you can write .javaimport to your current directory (project root
dictionary).

NOTE: configuring using .javaimport is deprecated. It will drops .javaimport
file format support in future release. Use |vim-javaclasspath| instead of
.javaimport, please.

an example for .javaimport below.
>
    # line comment
    {
        'path':    'path to classpath',
    },
    {
        'path':    'path to classpath',
        'javadoc': 'javadoc uri for classpath',
    },
<
this is like a JSON. for each dictionary must have "path" and may have
"javadoc". details below.

    "path" (required)
        possible values are
            - src directory path
            - jar file path
            - zip file path
            - javadoc url
        a value accepts absolute and relative path.

    "javadoc" (optional)
        javadoc url
